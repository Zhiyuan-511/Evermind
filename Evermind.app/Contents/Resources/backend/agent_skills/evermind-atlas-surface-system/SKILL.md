ATLAS SURFACE SYSTEM

Use this skill when a website risks looking flat, overly black/white, or visually unfinished.

Core rules:
- Build every page with at least 3 surface layers: base background, elevated panels, and accent glow/mist.
- Do not ship a page whose full background is only pure `#000`, pure `#fff`, or one flat solid fill.
- Define a reusable surface token set with CSS variables for: `--bg-base`, `--bg-elevated`, `--bg-contrast`, `--surface-border`, `--accent`, `--accent-soft`.
- Add one subtle atmospheric treatment per page: radial glow, grain/noise overlay, line pattern, or tinted gradient wash.
- Keep text/background contrast stable; decorative atmosphere must never reduce readability.
- Let hero, content sections, and footer shift tonally so the page has rhythm instead of one continuous slab.

Background composition patterns:
- Editorial dark: deep base + cool mist + warm accent edge light.
- Soft luxury light: warm paper base + tinted cards + faint noise or grid.
- Travel / cinematic: horizon gradient + ambient vignette + translucent surfaces.

Do not:
- stack 4 loud gradients with no hierarchy
- use the same background treatment for every section
- hide weak layout behind excessive blur/glow

Before finishing, verify:
- the page still reads clearly without relying on the hero image
- cards and nav remain legible over the background
- the design still feels premium if remote images fail

DOCS CLARITY ARCHITECTURE

Good documentation reduces ambiguity before it reduces word count.

- Start with purpose, audience, and expected outcome.
- Use stable sections: summary, prerequisites, steps, edge cases, troubleshooting.
- Prefer examples and checklists over abstract advice.
- Put the critical path first; move nuance and alternatives later.
- If a reader cannot tell what to do next within 10 seconds, rewrite the structure.

DIAGRAM-DRIVEN EXPLAINER

For docs, slides, and technical explainers, structure beats decoration.

- Convert dense concepts into 1 diagram + 3 supporting points whenever possible.
- Prefer timelines, flows, comparison grids, architectures, or state machines over long paragraphs.
- If external diagram tooling is unavailable, render with clean HTML/CSS/SVG blocks.
- Every diagram needs a one-line conclusion: what the viewer should notice.
- Avoid diagrams that merely mirror headings without adding understanding.

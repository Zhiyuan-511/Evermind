GODOGEN PLAYABLE LOOP

Do not build a "game concept". Build a playable loop first.

- Define the minimum playable loop: spawn, input, feedback, fail/win, restart.
- Split work into vertical-slice milestones: start screen, first interaction, enemy/obstacle, scoring, failure state, polish.
- Require evidence at each checkpoint: screenshot, browser/gameplay state change, score change, or movement proof.
- Keep scenes/components modular so a later agent can tune difficulty without rewriting the whole game.
- If QA cannot actually play the loop, the build is not approved.
- When revising, prioritize responsiveness and clarity before adding extra content.

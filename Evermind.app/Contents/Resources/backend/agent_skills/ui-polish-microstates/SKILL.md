UI POLISH MICROSTATES

Commercial quality often lives in state changes.

- Define hover, focus, active, loading, empty, success, and error states for obvious controls.
- Inputs should always reveal validation and focus clearly.
- Buttons should acknowledge click intent even when work continues asynchronously.
- Empty states should still teach the user what to do next.
- If the happy path looks polished but the edge states look generic, the product is not finished.

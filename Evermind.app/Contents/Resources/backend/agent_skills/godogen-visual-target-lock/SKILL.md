GODOGEN VISUAL TARGET LOCK

Do not let a 3D game drift between unrelated looks. Lock one target first.

- Choose one screenshot-grade target for the slice: camera framing, hero silhouette, enemy silhouette, weapon family, arena landmarks, HUD tone, and lighting direction.
- Keep the whole game inside one shape/material language. Do not mix military realism, neon toy props, and random low-poly graybox pieces in the same playable slice.
- Hero, enemies, weapons, pickups, cover props, muzzle flashes, damage indicators, and HUD accents must feel like they belong to the same game.
- If upstream imagegen/asset nodes provide a style lock, visual target, or manifest, treat that as the canonical source instead of improvising a new look.
- Readability beats decoration: the player silhouette, enemy threat read, weak points, and objective landmarks must still read at gameplay distance.
- For placeholder-era 3D, use deliberate stand-ins: layered materials, decals, emissive accents, proportions, and silhouettes that communicate the intended final asset class.
- Primitive geometry is acceptable only if it already reads as a believable proxy. Unlabeled cubes, mismatched colors, and random default materials are not an acceptable "finished look".
- A single stray Extrude/Tube/Lathe accent does not rescue a sphere/box/cylinder-dominated hero asset. The main player, enemy, and weapon masses must themselves read as authored silhouettes.
- During QA, reject any slice where the player cannot instantly distinguish hero, enemy, weapon, and navigable cover from one screenshot.
